Theory
Linux is an open source operating system (OS) created in 1991 by Linus Torvalds. Today, thanks to
its global community of enthusiasts, you can find it in all kinds of devices, from retail point-of-sale
systems to the world's most powerful supercomputers.
What is an operating system?
An OS is the software that directly manages a system’s hardware and resources, like the CPU,
memory, and storage. The OS sits between applications and hardware and makes the connections
between all your software and the physical resources that do the work.
Humans can interact with computers in many ways. Most people’s primary interaction with their
hardware happens through an OS, which helps them access a computer’s core functions. There are
lots of OS options, from proprietary software made by large companies to open source projects
created and supported by volunteers.
Think about an OS like a car engine. An engine can run on its own, but it becomes part of a
functional car when it’s connected with a transmission, axles, and wheels. Without the engine
running properly, the car won’t work.

In Linux, these commands are used to navigate the file system and view its structure. Below is an
explanation of each command and the specific directories mentioned.
Core Commands
• pwd (Print Working Directory): Displays the full absolute path of your current location.
• ls (List): Shows the files and subdirectories within the current directory.
• cd (Change Directory): Used to move from one directory to another.
Directory Overview
The directories mentioned are standard top-level folders in the Linux Filesystem Hierarchy Standard
(FHS):
• / (Root): The base of the entire file system; every other folder is located under this single
slash.
• /etc: Contains system-wide configuration files that control how programs and services
behave.
• /home: Houses the personal folders for individual users (e.g., /home/username), where they
store personal files and settings.
• /var (Variable): Stores data that changes frequently while the system is running, such as
system logs, caches, and spool files.
Sequential Breakdown of Your Input
If you run these commands in sequence, you would be performing the following actions:
1. pwd: See where you are starting (likely your user home).
2. ls /: List everything at the very top of the system (root).
3. cd /etc: Move into the configuration directory.
4. ls: View the configuration files inside /etc.
5. cd /home: Move into the user home parent directory.
6. ls: See a list of all user account folders on the system.
7. cd /var: Move into the variable data directory.
8. ls: View log folders and other frequently changing system data

Implementation of file system permissions and access control lists
DATE (ACLs) :
AIM
 To implement file permissions and ACLs in Linux.

Procedure
➢ Create a file.
➢ Assign permissions using chmod.
➢ Set ACL permissions using setfacl.
.
Core Commands
Implementation of Traditional File Permissions
Traditional file permissions are widely implemented in Unix-like systems (such as Linux) and focus
on three user classes: the owner, the owning group, and all others.
• Permissions: There are three basic permission types:
o Read (r): Allows viewing the content of a file or listing the contents of a directory.
o Write (w): Allows modifying a file's content or adding/removing files within a
directory.
o Execute (x): Allows running an executable file or script, or accessing a directory
(search permission).
• Commands (Linux):
o ls -l: Displays file permissions. A hyphen (-) indicates a regular file, and d indicates a
directory.
o chmod: Changes permissions using symbolic (e.g., u+r) or numeric (octal, e.g., 755)
modes.
o chown: Changes the file owner.
o chgrp: Changes the group owner.
Implementation of Access Control Lists (ACLs)
ACLs are an extension of the traditional permission model, allowing administrators to define
permissions for individual users or groups not covered by the standard owner and group
permissions.
• Key Concepts:
o Granularity: ACLs offer more detailed control, useful in complex environments where
specific access is needed for multiple users or groups.
o Access Control Entries (ACEs): Each ACL is a list of entries, with each ACE
specifying a user or group and their allowed/denied operations.
o ACL Mask: An ACL mask defines the maximum permissions that can be granted to
named users, named groups, and the owning group, ensuring compatibility with
applications that only understand traditional permissions.
o Default ACLs: These can be set on a directory to ensure that new files and
subdirectories created within it automatically inherit specific ACL entries.
o Inheritance: In some systems like Windows, ACLs have robust inheritance features,
while in Linux, default ACLs are used for new items.
• Commands (Linux): The primary command-line tools for managing POSIX ACLs
are getfacl and setfacl.
o getfacl [filename]: Displays the ACLs for a file or directory. A file with an ACL will
have a + symbol at the end of its permission string when using ls -l.
o setfacl -m u:[user]:[permissions] [filename]: Modifies the ACL to grant specific
permissions to a named user.
o setfacl -x u:[user] [filename]: Removes a specific user's ACL entry.
o setfacl -b [filename]: Removes all extended ACL entries.
o setfacl -d -m g:[group]:[permissions] [directory]: Sets a default ACL for a directory.

Monitoring and management of Linux system processes and
DATE resources :
AIM
 To monitor system processes and resource usage.
Procedure
1. Display running processes.
2. Monitor CPU and memory usage.
3. Terminate a process.

.
Core Commands
1. Process Monitoring (Real-Time & Snapshots)
• top: The standard real-time tool for viewing processes, CPU usage, and memory.
Press Shift+P to sort by CPU usage.
• htop: A modern, interactive upgrade to top with a color-coded interface and mouse support.
• ps: Provides a static snapshot of current processes. Use ps aux to see all running processes
across all users.
• pstree: Displays processes in a tree-like hierarchy to show parent-child relationships.
• pgrep: Searches for processes by name and returns their Process IDs (PIDs).
2. Process Management (Signals & Priority)
• kill: Terminates a process by its PID. kill -9  sends a SIGKILL to force termination.
• pkill / killall: Terminates processes by name rather than PID.
• nice: Starts a new process with a specific priority (niceness). Values range from -20 (highest
priority) to 19 (lowest).
• renice: Changes the priority of an already running process.
3. Resource-Specific Monitoring
• Memory: free -h shows a summary of physical and swap memory in human-readable format.
• Disk I/O: iostat -xz 1 monitors disk throughput and performance, while iotop identifies
specific processes hogging disk bandwidth.
• Network: ss -tuln (modern replacement for netstat) displays active listening ports and
connections.
• Overall System Stats: vmstat 1 reports on virtual memory, CPU activity, and system calls
every second.
4. Background & Job Control
• command &: Runs a command immediately in the background.
• jobs: Lists background jobs in the current shell session.
• fg %: Brings a background or stopped job to the foreground.
• bg %: Resumes a suspended job in the background.

Development of a shell script to display specific line ranges from a
DATE : given file
AIM
 To display specific line ranges from a given file using a shell script.
Procedure
A shell script to display specific line ranges from a file can be developed using standard Unix/Linux
command-line utilities such as sed, awk, or head and tail.
Here are three methods:
Method 1: Using sed (Recommended for simple line ranges)
The sed command uses the syntax sed -n 'start_line,end_linep' filename to print (the p flag) only the
lines within the specified range (the -n option suppresses default output).
bash
#!/bin/bash
# Check if file, start, and end lines are provided
if [ "$#" -ne 3 ]; then
 echo "Usage: $0   "
 exit 1
fi
FILE="$1"
START="$2"
END="$3"
# Check if the file exists
if [ ! -f "$FILE" ]; then
 echo "Error: File not found - $FILE"
 exit 1
fi
echo "--- Displaying lines $START to $END from $FILE ---"
# The 'p' command prints the selected lines
sed -n "${START},${END}p" "$FILE"
Method 2: Using awk (Good for flexible logic)
The awk command can check the current line number (NR) against the specified range.
bash
#!/bin/bash
# Check if file, start, and end lines are provided
if [ "$#" -ne 3 ]; then
 echo "Usage: $0   "
 exit 1
fi
FILE="$1"
START="$2"
END="$3"
# Check if the file exists
if [ ! -f "$FILE" ]; then
 echo "Error: File not found - $FILE"
 exit 1
fi
echo "--- Displaying lines $START to $END from $FILE ---"
# The condition '(NR >= START) && (NR <= END)' prints lines in range
awk -v start="$START" -v end="$END" 'NR >= start && NR <= end' "$FILE"
Method 3: Using head and tail (Best if only the end of the range is needed)
This method first uses head to get the first end_line lines, and then uses tail to select the
last (end_line - start_line) + 1 lines from that subset.
bash
#!/bin/bash
# Check if file, start, and end lines are provided
if [ "$#" -ne 3 ]; then
 echo "Usage: $0   "
 exit 1
fi
FILE="$1"
START="$2"
END="$3"
LINES_TO_DISPLAY=$((END - START + 1))
# Check if the file exists
if [ ! -f "$FILE" ]; then
 echo "Error: File not found - $FILE"
 exit 1
fi
echo "--- Displaying lines $START to $END from $FILE ---"
# Pipe the output of head to tail
head -n "$END" "$FILE" | tail -n "$LINES_TO_DISPLAY"
How to Use the Scripts:
1. Save one of the scripts above as a file (e.g., display_range.sh).
2. Make it executable from your terminal:
bash
chmod +x display_range.sh
3. Run it, providing the filename, start line, and end line as arguments:
bash
./display_range.sh your_file.txt 10 25
.

Database File Format
Each record is stored in the format:

Example:

Shell Script
#!/bin/bash
DB="student_db.txt"
touch "$DB"
while true
do
 echo "-----------------------------------"
 echo " Student Record Management System "
 echo "-----------------------------------"
 echo "1. Add Student Record"
 echo "2. View Student Records"
 echo "3. Delete Student Record"
 echo "4. Exit"
 echo "-----------------------------------"
 read -p "Enter your choice: " choice
 case $choice in
 1)
 read -p "Enter Roll Number: " roll
 read -p "Enter Name: " name
 read -p "Enter Department: " dept
 read -p "Enter Year: " year
 echo "$roll | $name | $dept | $year" >> "$DB"
 echo "Record added successfully."
 ;;
 2)
 if [ ! -s "$DB" ]; then

 echo "Student Records:"

 if grep -q "^$roll" "$DB"; then
