Installation and configuration of Red Hat Linux including an analysis of the booting process

🎯 Aim of the Experiment
To install and configure Red Hat Linux on a computer system and to analyze the Linux booting process, including the roles of BIOS/UEFI, bootloader, kernel, and system initialization.
⚙️ Hardware & Software Requirements
Computer system / Virtual Machine, Red Hat Enterprise Linux ISO, Bootable USB / VirtualBox, Minimum 4 GB RAM, 20 GB disk space
📚 Theoretical Background Overview
Linux is an open source operating system (OS) created in 1991 by Linus Torvalds. An OS is the software that directly manages a system’s hardware and resources, like the CPU, memory, and storage. The OS sits between applications and hardware and makes the connections between all your software and the physical resources that do the work.

The Linux Booting Process consists of several crucial phases:
1. BIOS/UEFI: Performs POST (Power-On Self-Test) and initializes hardware. It reads the MBR (Master Boot Record) or EFI System Partition to locate the bootloader.
2. Bootloader (GRUB2): Loads the Linux Kernel and initial RAM disk (initramfs) into memory.
3. Kernel: Initializes core hardware, mounts the root filesystem as read-only, and executes the first user-space process (`/sbin/init` or `systemd`).
4. System Initialization (systemd): Starts essential system services, target units (multi-user or graphical), and presents the login screen.
📋 Step-by-Step Procedure Execution
1. Select Install Red Hat Enterprise Linux in the boot menu and press Enter.
2. After Anaconda starts, select your language and region, and click Continue.
3. On the Installation Summary screen, configure Date & Time, Keyboard Layout, and Installation Destination (Automatic Partitioning).
4. Click Network & Hostname, toggle the Ethernet switch to connect to the network, and set a custom hostname.
5. Click Begin Installation.
6. While installing, set the Root Password and create a standard local user account under User Creation (optionally checking 'Make this user administrator').
7. Once installation finishes, click Reboot and log into your new environment.

#!/bin/bash
# permission_check.sh - Access permissions compliance audit system

read -p "Enter target directory path to run permission audit: " target_dir

#!/bin/bash
# student_db.sh - Student Record Management System Flat-File Application

DB_FILE="student_database.txt"
touch "$DB_FILE" # Ensure database container exists

while true; do
    echo "====================================================="
    echo "      STUDENT REGISTRY RECORD DATABASE MANAGER       "
    echo "====================================================="
    echo "1. Insert New Student Profile Entry"
    echo "2. Generate Detailed Tabular Registry View"
    echo "3. Remove Existing Student Record Entry"
    echo "4. Shutdown Application Environment"
    echo "====================================================="
    read -p "Select Menu Directive Option [1-4]: " option

    case $option in
        1)
            echo -e "
[ADD PROFILE INTERFACE]"
            read -p "Enter Target Student Unique Roll Number: " roll
            if grep -q "^$roll:" "$DB_FILE"; then
                echo "System Alert: Data conflict detected. Roll index already mapped."
            else
                read -p "Enter Candidate Full Legal Name: " name
                read -p "Enter Academic Department Branch: " dept
                read -p "Enter Evaluated Examination Score: " score
                # Save entry to file using colon delimiters
                echo "$roll:$name:$dept:$score" >> "$DB_FILE"
                echo "Profile registered successfully inside core ledger storage node."
            fi
            echo ""
            ;;
        2)
            echo -e "
[CURRENT RECORED REGISTRY VIEWER]"
            if [ ! -s "$DB_FILE" ]; then
                echo "System State Message: Storage arrays report empty dataset registries."
            else
                echo -e "ROLL NO\t| STUDENT NAME\t\t| DEPT\t| MARKS"
                echo "--------------------------------------------------------"
                # Process structural segments via field delimiters
                awk -F: '{printf "%-7s | %-20s | %-5s | %-5s\n", $1, $2, $3, $4}' "$DB_FILE"
            fi
            echo ""
            ;;
        3)
            echo -e "
[DELETION REMOVAL ENGINE]"
            read -p "Enter Targeted Unique Roll Number for Deletion: " roll
            if grep -q "^$roll:" "$DB_FILE"; then
                # Filter out the matching row and write the rest back
                grep -v "^$roll:" "$DB_FILE" > temp_db.txt
                mv temp_db.txt "$DB_FILE"
                echo "Target record removed from memory ledger nodes."
            else
                echo "System Entry Error: Selected index parameters do not match records."
            fi
            echo ""
            ;;
        4)
            echo "Shutting down application memory interfaces. Session complete."
            exit 0
            ;;
        *)
            echo "Selection mismatch. Pick alternative options within defined parameters."
            echo ""
            ;;
    esac
done
